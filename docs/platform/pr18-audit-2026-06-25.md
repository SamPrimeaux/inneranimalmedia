## 2. created_at datetime type — CRITICAL BUG FIXED

**Finding:** Both `agentsam_usage_events.created_at` and `agentsam_command_run.created_at` are `INTEGER NOT NULL DEFAULT unixepoch()` — unix epoch seconds. Confirmed via PRAGMA and live data sample (values ~1782430000).

**Original bug in `costs_report`:**
```python
# WRONG — produces TEXT timestamp; comparison against INTEGER is always false
base_where = f"created_at >= datetime('now', '{sqlite_modifier}')"
```

This caused `costs report --last 7d` to return zero rows for every window.

**Fix applied:**
```python
# CORRECT — returns INTEGER seconds, comparisons work
base_where = f"created_at >= unixepoch('now', '-{n} {unit}')"
```

Also added `_usage_events_where()` helper with a docstring explaining the type contract so future contributors don't re-introduce the bug.

`RECENT_SUSPICIOUS_RUNS_QUERY` now projects `datetime(created_at, 'unixepoch') AS created_at_utc` alongside the raw integer for readability in doctor reports.

## 5. Missing executor targets — schema mismatch corrected

**Original CLI check:**
```sql
(router_type = 'tool' AND COALESCE(tool_key, '') = '')
```

**Finding:** 85 active `router_type='tool'` commands have `tool_key=NULL`. These are intentional — they are wrangler-proxy and wrangler-terminal commands where `mapped_command` carries the identity (e.g. `wrangler d1 list`, `wrangler r2 bucket delete <NAME>`). The original check would flag all 85 as broken, which is a false positive.

**Fix applied:** Split into two checks:
- `missing_executor_targets` — only flags `workflow` with null `workflow_key` and `script` with null `tool_key`/`slug` (genuine breakage)
- `tool_routed_null_tool_key` — new informational check that lists tool-routed commands without a tool_key, for operator review; not treated as failures

## 6. Undocumented router_type values

The doctrine defines: `tool`, `workflow`, `script`, `in_app`. Live schema has:

| router_type | n |
|---|---|
| tool | 569 |
| agent | 26 |
| workflow | 7 |
| terminal_builtin | 3 |
| script | 3 |
| in_app | 3 |
| skill | 2 |

`agent`, `terminal_builtin`, and `skill` are not in the doctrine. All 26 `agent` rows are `risk_level=low` with no approval/confirmation required — benign. But these types are invisible to the original CLI's `missing_executor_targets` check.

**Fixes applied:**
- Added `KNOWN_ROUTER_TYPES` constant to the CLI for future guard usage
- Added `router_type_distribution` query to the doctor output so these show up every run
- Doctrine doc should be updated separately to add `agent`, `terminal_builtin`, `skill`
