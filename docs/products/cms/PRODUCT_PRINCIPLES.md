# CMS — product principles

---

## Principles

### 1. Federated content — unified shell

Client content stays on client Workers/D1/R2. IAM provides operator hub and editor GUI.

### 2. Do not bulk-import client DBs into IAM D1

Empty IAM `cms_pages` for client slugs is **expected**.

### 3. Familiar editor UX

Shopify-like tabs (Overview, Content, Theme, Store, Templates) — CF under the hood.

### 4. Agent Sam as co-pilot

Page edit, publish, bootstrap — not replacement for working WYSIWYG loop.

### 5. Standalone intent

**Yes** — resale as IAM CMS product for agencies and clients.
