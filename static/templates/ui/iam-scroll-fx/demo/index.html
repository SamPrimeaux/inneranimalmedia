Reusable scroll components · zero dependencies

One number drives every effect on this page.

Every animation below is a pure function of a single --progress custom property, written once per frame by scroll-engine.js. Nothing is triggered, nothing has hidden state — scrub up or down and it unwinds perfectly, because it's recomputed, not replayed.

01 · Primitivemask-wipe.css

--enter sweeps 0%→200% while --leave trails exactly 100% behind it. The 100%-wide gap between them is a soft edge that travels across the element and off the far side — works at any element width, no JS beyond writing the float. Swap the gradient angle for direction; .mask-wipe--iris gives a radial "open" instead. 

.mask-wipe {
  --enter: calc(var(--progress) * 200%);
  --leave: calc(var(--enter) - 100%);
  mask-image: linear-gradient(90deg, #000 var(--leave), transparent var(--enter));
}

Type that arrives with weight.

Three axes move at once — position, rotation, and tilt — which is the whole difference between this feeling expensive and feeling like a fade.

02 · Primitiveletter-stagger.css + split-text.js

splitText() wraps each character in a .ch span with an incremental --d delay. An IntersectionObserver adds .in once, and CSS transitions do every bit of the interpolation — no tween library, no per-letter JS on scroll. 

[data-split] .ch {
  transform: translateY(80px) rotateX(-60deg) rotateY(-20deg) rotate(-10deg);
  transition: transform 1s var(--ease-house) var(--d);
}
[data-split].in .ch { opacity: 1; transform: none; }

03 · Primitiveprogress-primitives.css

Bars use transform: scaleX(var(--progress)) so the browser compositor handles it, not layout. The frame uses clip-path: inset() driven by the same variable — swap .clip-reveal for --bottom-up or --center to change which edge grows from. 

.progress-bar::after { transform: scaleX(var(--progress)); }
.clip-reveal { clip-path: inset(0 calc((1 - var(--progress)) * 100%) 0 0); }

04 · Primitiveprogress-primitives.css

Cheap depth without a 3D engine. Each layer sets its own --depth (0 = fixed, 1 = full travel); one .parallax class computes a translateY offset from the shared --progress. Composited transform only — no reflow. 

.parallax {
  transform: translateY(calc((var(--progress) - 0.5) * var(--depth) * -140px));
}

The pin mechanism

A "chapter" is a tall track (height: var(--track-h)) with a position: sticky; top: 0; height: 100vh child. The child appears frozen while the track's extra height scrolls behind it.

Pin duration = track height − 100vh. No pin-spacer div, no JS-driven position changes, no jank on pin/unpin — the browser was always going to do this for free.

<section class="chapter" data-chapter style="--track-h: 300vh">
  <div class="chapter__sticky">
    ...
  </div>
</section>

<script>
  ScrollEngine.init({ selector: '[data-chapter]' });
</script>

Two clocks, not one

Scroll progress should drive narrative position. Anything meant to feel alive — idle motion, ambient drift — should run on its own clock so the page doesn't go rigid the instant the user stops scrolling.

This library only ships the scroll clock; wire a requestAnimationFrame ambient loop separately if you need one, and keep the two from feeding the same variable.

Reduced motion

Every primitive here has a prefers-reduced-motion: reduce fallback baked in — chapters collapse to normal flow, masks and clips clear, letters render fully formed. ScrollEngine detects it once at init and skips the frame loop entirely rather than animating to a snap.
